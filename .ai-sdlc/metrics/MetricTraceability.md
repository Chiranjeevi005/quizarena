# Metric Traceability Matrix

This matrix establishes the unbroken chain of custody from worker actions to continuous improvement outcomes.

## Traceability Path

```text
Worker --> Artifact --> Metric --> Dashboard --> Improvement
```

1. **Worker:** The AI persona executing a task (e.g., Implementation Engineer).
2. **Artifact:** The tangible output of the worker's effort (e.g., Pull Request, Implementation Plan).
3. **Metric:** The objective measurement extracted from the artifact (e.g., `M-IMPL-005` Lint Success Rate).
4. **Dashboard:** The visual aggregation of metrics for analysis.
5. **Improvement:** The human-approved modification to the SDLC based on dashboard trends.

## Example Trace
- **Worker:** Architecture Reviewer
- **Artifact:** `ArchitectureReview.md`
- **Metric:** `M-ARCH-001` (Pass Rate)
- **Dashboard:** Architecture Grouping
- **Improvement:** Update `ArchitectureStandards.md` to clarify a frequently misunderstood boundary rule.
